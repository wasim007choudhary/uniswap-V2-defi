# 9.4 — The Inflation Attack & Why `MINIMUM_LIQUIDITY` Exists

## 9.4A.1 — Can LP Tokens Become More Valuable?

---

# 🎯 Goal

Before understanding the Inflation Attack, we first need to answer a much simpler question.

> **Can someone intentionally increase the value represented by each LP Token?**

Notice this is **not** asking:

- Can someone steal funds?
- Can someone manipulate prices?
- Can someone exploit the AMM?

Instead, we're asking:

> **Can someone make each LP Token represent more assets than before?**

This question turns out to be the foundation of the entire Inflation Attack.

---

# 📌 What Does An LP Token Represent?

Earlier we proved that LP Tokens are **ownership receipts**.

They are **not** assets themselves.

Instead, they represent ownership of everything inside the Pair contract.

For example...

Suppose a pool contains

```text
100 ETH

100,000 USDC
```

and there are

```text
100 LP Tokens
```

in existence.

Each LP Token therefore represents approximately

```text
1%
```

ownership of the entire pool.

---

# 🧠 Important Observation

The value represented by one LP Token depends on two things:

1. The total value inside the pool.
2. The total LP Token supply.

Conceptually, you can think of it as

```text
Value of One LP Token

=

Pool Value

/

LP Supply
```

This is **not** an equation used by Uniswap.

It is simply a helpful mental model.

From this intuition we immediately notice:

If

```text
Pool Value

↑
```

while

```text
LP Supply

stays the same
```

then

```text
Each LP Token

becomes more valuable.
```

---

# 💬 Our Discussion

At this point we asked an important question.

> **Can someone increase the pool's value without increasing the LP Token supply?**

Normally this seems impossible.

Whenever liquidity is added:

- the pool becomes larger,
- new LP Tokens are minted,
- ownership remains proportional.

Everything stays balanced.

---

# But There Is Another Way

One important property of ERC20 tokens is that anyone can transfer them directly.

For example,

```solidity
token.transfer(pair, amount);
```

does **not** require calling

```solidity
pair.mint(...)
```

As a result,

tokens can enter the Pair contract without any new LP Tokens being created.

---

# Example

Suppose Alice creates a pool containing

```text
100 ETH

100,000 USDC
```

Earlier we derived that the initial LP Supply is

```text
√(100 × 100,000)

≈3162 LP Tokens
```

Assume Alice currently owns every LP Token.

Ownership:

```text
Alice

↓

100%
```

Now Alice simply transfers

```text
1,000,000 USDC
```

directly into the Pair contract.

Importantly,

she does **not** call

```solidity
mint()
```

---

# What Changed?

The pool now contains significantly more assets.

However...

The LP Supply is completely unchanged.

Before:

```text
Pool Value

↓

Smaller
```

```text
LP Supply

↓

3162
```

After:

```text
Pool Value

↓

Much Larger
```

```text
LP Supply

↓

3162
```

---

# 🧠 Huge Realization

Returning to our mental model,

```text
Value of One LP Token

=

Pool Value

/

LP Supply
```

Since

```text
Pool Value

↑
```

while

```text
LP Supply

does not change
```

every LP Token now represents significantly more value than before.

Without minting a single new LP Token,

Alice has increased the value represented by each existing LP Token.

---

# 💬 Our Discussion

At this point an important question naturally arose.

You asked:

> **"When she added extra to the pool, didn't she use her own money? And arbitrage guys came to take profits which returns the pool to normal right? Then she lost money though in that time?"**

This question led us to an important realization.

---

# Did Alice Use Her Own Money?

Yes.

The donated USDC came directly from Alice's wallet.

Those tokens genuinely left her possession.

---

# Did Alice Become Richer?

No.

This is a very common misconception.

Alice simply moved value from

```text
Alice's Wallet
```

into

```text
Alice's LP Position.
```

Since she currently owns

```text
100%
```

of the LP Tokens,

she also owns

```text
100%
```

of the Pair.

Her total wealth remains approximately unchanged.

She has simply changed **where** that wealth is stored.

---

# 💬 Our Discussion

Initially we started discussing arbitrage here.

The reasoning was:

- Alice donates USDC.
- The pool price changes.
- Arbitrage traders trade against the pool.
- Eventually the market price is restored.

While this is true in practice...

we realized something important during our discussion.

---

# Why We Temporarily Ignore Arbitrage

The Inflation Attack is **not fundamentally about price manipulation.**

Instead, it is about:

- LP Token accounting,
- LP Supply,
- LP Token value,
- and integer rounding during minting.

Arbitrage is simply a real-world consequence of changing the reserve ratio.

It is **not** the mechanism that makes the Inflation Attack possible.

To avoid mixing two separate ideas,

we intentionally postpone arbitrage until the end of the chapter.

---

# 🧠 Biggest Realization

By the end of this discussion we had ruled out our first incorrect hypothesis.

The attack is **not**

```text
Donate

↓

Immediately Profit
```

Instead,

the donation merely prepares the pool.

Alice has not exploited anyone.

She has simply increased the value represented by each LP Token.

The actual exploit, if one exists, can only occur when someone else later interacts with the pool.

---

# Questions We Resolved

### ❓Can someone increase the value represented by each LP Token?

Yes.

By increasing the assets inside the Pair without minting additional LP Tokens.

---

### ❓Does donating assets immediately make the attacker richer?

No.

The attacker simply moves wealth from their wallet into their LP position.

---

### ❓Is arbitrage the Inflation Attack?

No.

Arbitrage is a separate market mechanism.

The Inflation Attack is fundamentally an LP accounting and integer-rounding problem.

---

# Common Misconceptions

### ❌ Donating assets is itself the attack.

False.

The donation is only the setup.

---

### ❌ Alice profits immediately after donating.

False.

Her overall wealth is approximately unchanged.

---

### ❌ The attack is about manipulating the market price.

False.

The important issue is LP Token accounting, not temporary price changes.
