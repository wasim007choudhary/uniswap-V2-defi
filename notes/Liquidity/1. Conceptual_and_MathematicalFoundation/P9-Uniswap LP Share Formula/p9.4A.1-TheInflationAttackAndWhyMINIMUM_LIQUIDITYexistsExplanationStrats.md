# 9.4 — The Inflation Attack & Why `MINIMUM_LIQUIDITY` Exists

## 9.4A.1 — Can LP Tokens Become More Valuable?

---

# 🎯 Goal

Before understanding the Inflation Attack, we first need to answer a much simpler question.

> **Can someone intentionally increase the value represented by each LP Token?**

Notice this is **not** asking:

- Can someone steal funds?
- Can someone manipulate prices?
- Can someone exploit the AMM?

Instead, we're asking:

> **Can someone make each LP Token represent more assets than before?**

Understanding this idea is the foundation of the Inflation Attack.

---

# 📌 What Does An LP Token Represent?

Earlier we proved that LP Tokens are **ownership receipts**.

They are **not** assets themselves.

Instead, they represent ownership of everything inside the Pair contract.

Suppose a pool contains

```text
100 ETH

100,000 USDC
```

and there are

```text
100 LP Tokens
```

in existence.

Each LP Token therefore represents approximately

```text
1%
```

ownership of the pool.

---

# 🧠 Important Observation

The value represented by one LP Token depends on two things.

1. The value stored inside the pool.
2. The total LP Token supply.

A useful mental model is

```text
Value of One LP Token

=

Pool Value

/

LP Supply
```

This is **not** a Uniswap formula.

It is simply an intuition.

Immediately we can see:

If

```text
Pool Value

↑
```

while

```text
LP Supply

stays the same
```

then

```text
Each LP Token

becomes more valuable.
```

---

## 🤔 Natural Question

Can someone increase the pool value **without** increasing the LP supply?

Normally this seems impossible.

Whenever liquidity is added:

- Pool value increases.
- New LP Tokens are minted.
- Ownership stays proportional.

Everything remains balanced.

---

# 📌 There Is Another Way

Remember something we learned earlier.

ERC20 tokens can always be transferred directly.

For example,

```solidity
token.transfer(pair, amount);
```

does **not** require calling

```solidity
pair.mint(...)
```

This means tokens can enter the Pair contract without minting any new LP Tokens.

---

# 📌 Example

Suppose Alice creates a pool containing

```text
100 ETH

100,000 USDC
```

Earlier we derived the initial LP Supply as

```text
√(100 × 100,000)

≈3162
```

(We'll later refine this using real on-chain units.)

Assume Alice owns

```text
100%
```

of the LP Tokens.

Now Alice directly transfers

```text
1,000,000 USDC
```

to the Pair contract.

Importantly,

she **does not** call

```solidity
mint()
```

---

# What Changed?

Before:

```text
Pool Value

↓

Smaller
```

```text
LP Supply

↓

3162
```

After:

```text
Pool Value

↓

Much Larger
```

```text
LP Supply

↓

3162
```

The pool became larger.

The LP Supply stayed exactly the same.

---

# 🧠 Important Observation

Returning to our mental model,

```text
Value of One LP Token

=

Pool Value

/

LP Supply
```

Pool value increased.

LP Supply remained unchanged.

Therefore,

every LP Token now represents significantly more value than before.

Without minting a single additional LP Token,

Alice has made every existing LP Token more valuable.

---

# 💬 Your Question

At this point you asked:

> **"When she added extra to the pool, didn't she use her own money? And arbitrage guys came to take profits which returns the pool to normal right? Then she lost money though in that time?"**

This question completely changed the direction of our discussion.

---

# ✅ Did Alice Use Her Own Money?

Yes.

The donated USDC genuinely came from Alice's wallet.

Those tokens are no longer in her possession.

---

# 🤔 Then Did Alice Become Richer?

No.

This is one of the easiest misconceptions to make.

Alice simply moved wealth from

```text
Alice's Wallet
```

to

```text
Alice's LP Position.
```

Since she currently owns

```text
100%
```

of the LP Tokens,

she also owns

```text
100%
```

of the Pair contract.

Her total wealth remains approximately the same.

She merely changed **where** that wealth is stored.

---

# 🧠 Huge Realization

At this point we ruled out our first hypothesis.

The attack is **not**

```text
Donate

↓

Immediately Profit
```

Nothing has been exploited.

Nothing has been stolen.

The donation is only preparing the pool.

---

# 🤔 What About Arbitrage?

Initially we started reasoning like this:

```text
Donation

↓

Pool price changes

↓

Arbitrage traders profit

↓

Pool price returns to normal
```

This reasoning is correct.

In a real market,

arbitrage traders would eventually restore the pool price.

---

# 💡 Why We Temporarily Ignore Arbitrage

During our discussion we realized something important.

The Inflation Attack is **not fundamentally about price manipulation.**

Instead,

it is about

- LP Token accounting,
- LP Supply,
- LP Token value,
- and integer rounding.

Arbitrage is simply a market consequence of changing reserves.

It is **not** what creates the vulnerability.

Therefore,

to understand the attack clearly,

we intentionally ignore arbitrage until the end of the chapter.

---

# 💬 Your Next Question

You then asked:

> **"If I'm Alice and create the pool, am I in loss?"**

Excellent question.

The answer is:

**Technically, yes.**

But the actual cost is almost zero.

Let's see why.

---

# 📌 First Calculation (Human-Readable Units)

Suppose Alice creates a pool with

```text
10 ETH

10,000 USDC
```

Liquidity appears to be

```text
√(10 × 10,000)

≈316
```

If we subtract

```text
MINIMUM_LIQUIDITY = 1000
```

this makes no sense.

It looks like the transaction should fail.

Even for larger examples,

it looked like Alice might lose a huge percentage of the pool.

---

# 🚫 Common Mistake

The mistake is that we used

**human-readable token amounts.**

The Pair contract never sees

```text
10 ETH
```

It sees

```text
10 × 10¹⁸
```

because ETH uses 18 decimals.

Likewise,

it never sees

```text
10,000 USDC
```

It sees

```text
10,000 × 10⁶
```

because USDC has 6 decimals.

---

# 📌 Recalculating Using Real On-Chain Units

Now the contract computes

```text
√(
(10 × 10¹⁸)
×

(10,000 × 10⁶)
)
```

The result is roughly

```text
10¹⁵
```

LP Tokens.

Not

```text
316.
```

This is many orders of magnitude larger than

```text
1000.
```

---

# 📌 Real-Money Perspective

Suppose Alice creates a

```text
$20,000
```

pool.

Approximately

```text
10¹⁵
```

LP Tokens are created.

Only

```text
1000
```

are permanently locked.

Ownership sacrificed is roughly

```text
1000

/

10¹⁵
```

which is effectively

```text
0%.
```

The permanently locked value is so tiny that it is practically unnoticeable for honest pool creators.

---

# 🧠 Huge Realization

Earlier it looked like Alice sacrificed

30%

or more.

That conclusion was completely wrong because we ignored token decimals.

The contract performs every calculation using raw base units.

As a result,

```text
√(amount0 × amount1)
```

is usually astronomically larger than

```text
MINIMUM_LIQUIDITY = 1000.
```

The actual economic cost is negligible.

---

# Questions We Resolved

### ❓ Can someone increase the value represented by each LP Token?

Yes.

By increasing the assets inside the Pair without minting new LP Tokens.

---

### ❓ Does donating immediately make the attacker richer?

No.

The attacker simply moves wealth from their wallet into their LP position.

---

### ❓ Is arbitrage the Inflation Attack?

No.

Arbitrage is a separate market mechanism.

The Inflation Attack is fundamentally an LP accounting and integer-rounding problem.

---

### ❓ Is the first LP actually in significant loss because of `MINIMUM_LIQUIDITY`?

No.

Although some LP Tokens are permanently locked,

the fraction is calculated against the enormous LP supply created from raw token units.

The economic cost is effectively negligible.

---

# 🚫 Common Misconceptions

### ❌ Donating assets is the attack.

False.

It is only the setup.

---

### ❌ Alice profits immediately after donating.

False.

She simply changes where her wealth is stored.

---

### ❌ The attack is about changing the market price.

False.

It is fundamentally about LP accounting.

---

### ❌ The first LP loses a significant percentage of the pool.

False.

That conclusion comes from using human-readable token amounts instead of raw on-chain units.

---

# 🧠 Biggest Realizations

- LP Tokens become more valuable whenever pool value increases while LP Supply stays constant.
- Donations alone do **not** create profit.
- Alice simply moves wealth from her wallet into her LP position.
- The Inflation Attack is about LP accounting—not price manipulation.
- `MINIMUM_LIQUIDITY` costs almost nothing for honest users because all calculations use raw token units rather than human-readable values.
