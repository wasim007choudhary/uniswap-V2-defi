# `_mintFee()` — Chapter 7 (Part 2): Solving the Equation

In the previous chapter, we established the most important equation in the entire derivation.

```text
            x                (L₁ − L₀)/6
      ─────────────   =   ─────────────────
          S + x                 L₁
```

where:

```text
L₀ = Previous Liquidity = rootKLast

L₁ = Current Liquidity = rootK

S = Current LP Token Supply = totalSupply

x = LP Tokens To Mint
```

At this point, the difficult conceptual work is finished.

Everything that follows is simply algebra.

Our goal is to isolate:

```text
x
```

because that is exactly the number of LP tokens the protocol must mint.

------------------------------------------------------------------------

# Step 1 — Remove The Complex Fraction

The right-hand side contains a fraction divided by another fraction.

```text
    (L₁ − L₀)/6
────────────────────
        L₁
```

Although this is mathematically correct, it is easier to work with if we rewrite it as a single fraction.

Recall the rule:

```text
 a/b
──────
  c

=

 a
────
 bc
```

Applying that rule gives:

```text
   L₁ − L₀
─────────────
     6L₁
```

Therefore our equation becomes:

```text
         x        L₁ − L₀
      ──────  =  ─────────
       S + x        6L₁
```

Nothing changed mathematically.

We simply rewrote the expression into a cleaner form.

------------------------------------------------------------------------

# Step 2 — Cross Multiply

To eliminate the fractions, we cross multiply.

Multiply:

- the left numerator by the right denominator, and
- the right numerator by the left denominator.

This gives:

```text
6L₁x

=

(L₁ − L₀)(S + x)
```

At this point there are no fractions left.

------------------------------------------------------------------------

# Step 3 — Expand The Brackets

Now expand the right-hand side.

```text
(L₁ − L₀)(S + x)
```

Distribute:

```text
(A)(B + C)

=

AB + AC
```

Therefore:

```text
(L₁ − L₀)(S + x)

=

S(L₁ − L₀) + x(L₁ − L₀)
```

The equation now becomes:

```text
6L₁x

=

S(L₁ − L₀) + x(L₁ − L₀)
```

Notice something interesting.

The unknown:

```text
x
```

appears on **both sides** of the equation.

This is expected.

Remember why?

Because minting LP tokens changes:

```text
totalSupply
```

which changes ownership percentages.

The unknown naturally appears on both sides of the equation.

------------------------------------------------------------------------

# Step 4 — Move Every `x` To One Side

Subtract:

```text
x(L₁ − L₀)
```

from both sides.

This gives:

```text
6L₁x

−

x(L₁ − L₀)

=

S(L₁ − L₀)
```

Now every term containing:

```text
x
```

is on the left-hand side.

------------------------------------------------------------------------

# Step 5 — Factor Out `x`

Both terms on the left contain:

```text
x
```

so factor it out.

Exactly like:

```text
6a − 2a

=

a(6 − 2)
```

we obtain:

```text
x(6L₁ − (L₁ − L₀) ) = S(L₁ − L₀)
```

------------------------------------------------------------------------

# Step 6 — Simplify The Brackets

Now simplify:

```text
6L₁ − (L₁ − L₀)
```

Be careful here.

The minus sign applies to the **entire bracket**.

Distribute the minus sign:

```text
6L₁ − L₁ + L₀
```

Now combine like terms.

```text
6L₁ − L₁ = 5L₁
```

Therefore:

```text
6L₁ − (L₁ − L₀)

=

5L₁ + L₀
```

This is exactly where the famous:

```text
5 × rootK + rootKLast
```

comes from.

It is **not** an arbitrary constant.

It naturally appears during the algebraic simplification.

The equation now becomes:

```text
x(5L₁ + L₀) = S(L₁ − L₀)
```

------------------------------------------------------------------------

# Step 7 — Isolate `x`

Finally, divide both sides by:

```text
5L₁ + L₀
```

to isolate:

```text
x
```

We obtain:

```text
         S(L₁ − L₀)
x =   ───────────────
         5L₁ + L₀
```

This is the complete mathematical solution.

------------------------------------------------------------------------

# Replace The Symbols

Now substitute the variables back into their Solidity equivalents.

```text
S

↓

totalSupply

L₁

↓

rootK

L₀

↓

rootKLast

x

↓

liquidity
```

The equation becomes:

```text
               totalSupply(rootK − rootKLast)
liquidity = ────────────────────────────────────
                    5(rootK) + rootKLast
```

------------------------------------------------------------------------

# Compare It With The Solidity Code

Now compare the mathematical result with the implementation.

```solidity
uint256 numerator = totalSupply.mul(rootK.sub(rootKLast));

uint256 denominator = rootK.mul(5).add(rootKLast);

uint256 liquidity = numerator / denominator;
```

They match **exactly**.

Breaking it down:

The numerator:

```text
totalSupply(rootK − rootKLast)
```

becomes:

```solidity
uint256 numerator =
    totalSupply.mul(rootK.sub(rootKLast));
```

The denominator:

```text
5(rootK) + rootKLast
```

becomes:

```solidity
uint256 denominator =
    rootK.mul(5).add(rootKLast);
```

Finally:

```text
 Numerator
────────────
 Denominator
```

becomes:

```solidity
uint256 liquidity = numerator / denominator;
```

------------------------------------------------------------------------

# The Magic Formula Isn't Magic

When developers first encounter:

```solidity
uint256 numerator = totalSupply.mul(rootK.sub(rootKLast));
uint256 denominator = rootK.mul(5).add(rootKLast);
uint256 liquidity = numerator / denominator;
```

it often looks like a mysterious formula that someone invented.

After completing the derivation, we can see that this is **not** the case.

Nothing in the formula is arbitrary.

It is simply the unique mathematical solution to the following problem:

> **"How many ordinary LP tokens must be minted so that, after minting increases `totalSupply`, the protocol owns exactly one-sixth of the fee-generated liquidity growth?"**

Every term in the Solidity implementation follows naturally from solving that equation.

The formula is not magic.

It is mathematics.
