# 1. Introduction

Imagine there are two shops selling exactly the same phone.

```
Shop A

iPhone

₹90,000
```

```
Shop B

iPhone

₹95,000
```

Immediately you notice something.

The phone is cheaper in Shop A.

If you could somehow get the phone from Shop A first, sell it to Shop B, pay Shop A back, and keep the remaining money...

you would make a profit.

That is exactly what Arbitrage is.

---

Normally you cannot do this because you don't own the phone.

Likewise in DeFi:

You cannot sell WETH if you don't own WETH.

Flash Swaps remove this limitation.

Instead of buying WETH first...

the Pair lends it to you.

For free.

Well...

temporarily.

---

The Pair only asks one thing.

> Before this transaction ends...

> Pay me back.

If you fail...

everything is reverted as though nothing happened.

# 2. The Problem

Suppose two DEXes disagree.

```
Uniswap

1 WETH

=

1765 DAI
```

```
SushiSwap

1 WETH

=

1802 DAI
```

Exactly the same asset.

Different price.

This creates an Arbitrage opportunity.

Because WETH is cheaper on one exchange and more expensive on another.

---

But there is one problem.

You own

```
0 WETH
```

No WETH means no arbitrage.

Flash Swaps solve this.

# 3. What is Arbitrage?

Arbitrage simply means

```
Buy Cheap

↓

Sell Expensive

↓

Keep Difference
```

Nothing more.

Nothing less.

Every arbitrage strategy in DeFi is some variation of this.

The only thing that changes is

- where you buy
- where you sell
- what asset you use
- how you finance the trade

---

# 4. Why Flash Swaps Exist

Normally trading looks like this.

```
Wallet

↓

Own WETH

↓

Swap

↓

Receive DAI
```

Flash Swaps reverse the order.

```
Wallet

↓

Borrow WETH

↓

Swap

↓

Receive DAI

↓

Repay

↓

Keep Profit
```

Notice something.

We never needed initial capital.

The protocol temporarily became our lender.

---

# 5. Mental Model

Many beginners imagine this:

```
Borrow DAI

↓

Buy WETH

↓

Sell WETH

↓

Repay DAI
```

That is **NOT** what our contract does.

Our contract works like this.

```
Borrow WETH

↓

Sell WETH

↓

Receive DAI

↓

Repay Pair

↓

Keep Remaining DAI
```

Notice something important.

We never purchased WETH.

We borrowed it.

That borrowed WETH becomes our temporary inventory.

We immediately sell it.

# 7. Overall Transaction Flow

Our contract follows this sequence.

```
          Pair0
             │
             │
 Borrow WETH │
────────────▶│
             │
             ▼
       ARB Contract
             │
             │ Send WETH
             ▼
          Pair1
             │
             │
 Swap WETH→DAI
             │
             ▼
       ARB Contract
             │
             │ Repay Pair0
             ▼
          Pair0
             │
             ▼
        Keep Profit
```

Everything happens inside

ONE

single

Ethereum transaction.

No second transaction.

No waiting.

No risk of someone paying before you repay.

If repayment fails...

the entire transaction is reverted.

---

# 8. Why Borrow From Pair0?

Pair0 is the exchange where borrowing is cheapest for our strategy.

We are not choosing Pair0 randomly.

We intentionally choose the pool whose pricing allows us to borrow WETH and later repay with less DAI than we can obtain elsewhere.

Think of Pair0 as

```
Temporary Lender
```

It is NOT the place where we make profit.

It simply provides temporary liquidity.

---

# 9. Why Trade On Pair1?

Pair1 is where the opportunity exists.

Its price is different.

Our borrowed WETH is immediately sent here.

Here the WETH is sold.

In return we receive DAI.

This is where the arbitrage profit is generated.

Without Pair1 having a different price...

there is no arbitrage.

---

# 10. Why Repay Pair0?

Flash Swaps are not free.

The Pair expects to finish the transaction without losing value.

Before the transaction completes,

Pair0 checks

> "Have my reserves been restored according to the AMM invariant?"

If yes,

everything succeeds.

Otherwise,

the Pair reverts the entire transaction.

---

# 11. Why Can We Repay In Another Token?

This confuses almost everyone the first time.

We borrow

```
WETH
```

But repay

```
DAI
```

How?

Because Uniswap V2 does **not** require

```
Borrow WETH

↓

Return WETH
```

Instead it requires

```
Borrow WETH

↓

Restore pool value
```

The Pair only cares that its invariant is satisfied after the transaction.

If enough DAI enters the pool,

the Pair is happy.

It does not care that the borrowed asset was WETH.

This is one of the most important concepts to understand in Flash Swaps.

# 12. Contract Overview

Our contract contains only four major pieces.

```
ARB2
│
├── FlashSwap()
│
├── uniswapV2Call()
│
├── getAmountIn()
│
└── getAmountOut()
```

Each function has one responsibility.

**FlashSwap()**

Starts the flash swap.

---

**uniswapV2Call()**

The callback executed by Pair0 after sending us the borrowed tokens.

This is where the real arbitrage happens.

---

**getAmountIn()**

Calculates

> "How much DAI do I need to repay if I borrowed this much WETH?"

---

**getAmountOut()**

Calculates

> "If I sell this much WETH on Pair1...

how much DAI will I receive?"

---

Everything else in the contract exists to connect these four building blocks.
