# Step 9 – Solving for the Optimal Amount In

> **Goal of this step**
>
> In **Step 8**, we derived the derivative of the arbitrage output:
>
> ```
>                     xₐx_byₐy_b
> dy_b' = ---------------------------------------------
>           (x_byₐ + (xₐ + x_b)dyₐ)²
> ```
>
> Earlier in the derivation, we also proved that the profit function satisfies
>
> ```
> F'(dyₐ) = dy_b' - 1
> ```
>
> and that maximum profit occurs when
>
> ```
> F'(dyₐ) = 0
> ```
>
> Therefore,
>
> ```
> dy_b' = 1
> ```
>
> This step is simply an algebra exercise.
>
> We'll solve this equation to obtain the **Optimal Amount In**.

---

# Where We Are

From Step 8,

```
                    xₐx_byₐy_b
dy_b' = ---------------------------------------------
          (x_byₐ + (xₐ + x_b)dyₐ)²
```

At the point of maximum profit,

```
dy_b' = 1
```

Substitute this condition.

```
                    xₐx_byₐy_b
------------------------------------------------ = 1
       (x_byₐ + (xₐ + x_b)dyₐ)²
```

Everything except

```
dyₐ
```

is a constant.

Our objective is simply to isolate that variable.

---

# Step 1 – Remove The Denominator

Multiply both sides by

```
(x_byₐ + (xₐ + x_b)dyₐ)²
```

The denominator disappears.

Result

```
xₐx_byₐy_b

=

(x_byₐ + (xₐ + x_b)dyₐ)²
```

Notice something.

The right-hand side is simply a perfect square.

---

# Step 2 – Undo The Square

To isolate the expression,

take the square root of both sides.

```
√(xₐx_byₐy_b)

=

x_byₐ + (xₐ + x_b)dyₐ
```

Mathematically,

taking the square root normally produces both a positive and negative solution.

However,

a trade size cannot be negative.

Since we're solving for a valid arbitrage input,

we keep the positive root.

---

# Step 3 – Move The Constant Term

Subtract

```
x_byₐ
```

from both sides.

```
√(xₐx_byₐy_b)

-

x_byₐ

=

(xₐ + x_b)dyₐ
```

The unknown now appears in only one term.

---

# Step 4 – Divide By The Coefficient

Finally,

divide both sides by

```
xₐ + x_b
```

Result

```
                √(xₐx_byₐy_b) - x_byₐ
dyₐ = ------------------------------------------
                    xₐ + x_b
```

This is the closed-form solution.

It tells us exactly how much of the input token should be traded to maximize arbitrage profit.

---

# The Final Formula

The **Optimal Amount In** is

```
                √(xₐx_byₐy_b) - x_byₐ
dyₐ = ------------------------------------------
                    xₐ + x_b
```

There is no guessing,

no repeated simulations,

and no brute-force search.

Given the reserve values,

the optimal input can be calculated immediately.

---

# Why Does The Square Root Appear?

Many people are surprised to see a square root in the final equation.

It isn't a special property of Uniswap.

It appears naturally because the derivative equation contained a squared term.

Earlier,

we had

```
(x_byₐ + (xₐ + x_b)dyₐ)²
```

To isolate the variable,

we needed the inverse operation of squaring.

That inverse operation is the square root.

The square root is therefore a consequence of ordinary algebra,

not a unique feature of automated market makers.

## Step 7 — Simplify The Algebra

After simplifying the nested fraction,

```
                xₐy_bdyₐ
dy_b = -----------------------------------------
        x_byₐ + (xₐ + x_b)dyₐ
```

Now everything depends on only one variable,

```
dyₐ
```

---

## Step 8 — Differentiate

Using the Quotient Rule,

```
                    xₐx_byₐy_b
dy_b' = ---------------------------------------------
          (x_byₐ + (xₐ + x_b)dyₐ)²
```

---

## Step 9 — Solve

Set

```
dy_b'

=

1
```

Solve for

```
dyₐ
```

Result

```
                √(xₐx_byₐy_b) - x_byₐ
dyₐ = ------------------------------------------
                    xₐ + x_b
```

The derivation is complete.
