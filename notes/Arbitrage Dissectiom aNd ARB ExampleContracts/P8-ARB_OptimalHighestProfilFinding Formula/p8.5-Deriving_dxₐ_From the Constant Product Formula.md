# Step 5 ‚Äì Deriving `dx‚Çê` From the Constant Product Formula

> **Goal of this step**
>
> In **Step 4**, we discovered that the arbitrage output (`dy_b`) is **not** produced directly from our input (`dy‚Çê`).
>
> Instead, the dependency chain is
>
> ```
> dy‚Çê
> ‚Üì
> dx‚Çê
> ‚Üì
> dy_b
> ```
>
> Therefore, before we can determine `dy_b`, we must first answer a simpler question:
>
> > **"Given an input of `dy‚Çê`, how much `dx‚Çê` do we receive from AMM A?"**
>
> This is nothing more than the standard Uniswap swap calculation derived from the Constant Product Formula.

---

# Zoom Into AMM A

Forget AMM B for now.

We're only interested in the **first swap**.

Initially the pool contains

```
Reserve X = x‚Çê

Reserve Y = y‚Çê
```

As always,

the pool satisfies

```
x‚Çê ¬∑ y‚Çê = k
```

This is simply the Constant Product Formula we've already studied.

Nothing new has been introduced.

---

# We Perform A Swap

Suppose we send

```
dy‚Çê
```

into the pool.

Since these tokens are entering the pool,

the Y reserve increases.

```
New Y Reserve

=

y‚Çê + dy‚Çê
```

---

At the same time,

the pool gives us some X tokens.

Let's call that amount

```
dx‚Çê
```

Since these tokens leave the pool,

the X reserve decreases.

```
New X Reserve

=

x‚Çê - dx‚Çê
```

Notice the signs.

```
Input Token

+

because it enters the pool.
```

```
Output Token

-

because it leaves the pool.
```

Exactly the same reserve updates we've seen throughout Uniswap V2.

---

# The Constant Product Must Still Hold

After the swap,

the product of the new reserves must still equal the constant.

```
(New X Reserve)

√ó

(New Y Reserve)

=

k
```

Substitute the updated reserves.

```
(x‚Çê - dx‚Çê)

√ó

(y‚Çê + dy‚Çê)

=

k
```

But before the swap,

we already knew

```
x‚Çêy‚Çê

=

k
```

Since both expressions equal the same constant,

they must equal each other.

```
(x‚Çê - dx‚Çê)(y‚Çê + dy‚Çê)

=

x‚Çêy‚Çê
```

This equation is the starting point for deriving the swap output.

---

# Our Goal

We're **not** trying to solve for

```
k
```

or

```
x‚Çê
```

or

```
y‚Çê
```

Our goal is much simpler.

> **Solve for `dx‚Çê`.**

---

# Step 1 ‚Äì Expand The Left Side

Apply the distributive property.

Remember

```
(a - b)(c + d)

=

ac + ad - bc - bd
```

Applying that rule,

```
x‚Çêy‚Çê

+

x‚Çêdy‚Çê

-

dx‚Çêy‚Çê

-

dx‚Çêdy‚Çê

x‚Çêy‚Çê
```

Nothing special happened.

We simply multiplied every term.

---

# Step 2 ‚Äì Cancel Common Terms

Notice

```
x‚Çêy‚Çê
```

appears on both sides.

Subtract it from both sides.

We're left with

```
x‚Çêdy‚Çê

-

dx‚Çêy‚Çê

-

dx‚Çêdy‚Çê

=

0
```

Already much cleaner.

---

# Step 3 ‚Äì Move The `dx‚Çê` Terms Together

Move

```
x‚Çêdy‚Çê
```

to the opposite side,

or equivalently move the `dx‚Çê` terms to the other side.

The result becomes

```
x‚Çêdy‚Çê

=

dx‚Çêy‚Çê

+

dx‚Çêdy‚Çê
```

Notice something.

Both terms on the right contain

```
dx‚Çê
```

Exactly what we wanted.

---

# Step 4 ‚Äì Factor Out `dx‚Çê`

Just like

```

=

x(3 + 5)
```

we factor

```
dx‚Çê
```

from the right side.

```
x‚Çêdy‚Çê

=

dx‚Çê(y‚Çê + dy‚Çê)
```

Now the unknown appears only once.

---

# Step 5 ‚Äì Divide Both Sides

To isolate

```

```

divide both sides by

```
(y‚Çê + dy‚Çê)
```

Result

```
           x‚Çêdy‚Çê
dx‚Çê = ---------------
        y‚Çê + dy‚Çê
```

We have successfully derived the output amount of the first swap.

# Where Did The Trading Fee Go?

You might notice something missing.

Where are

The lesson intentionally ignores swap fees during the derivation.

Without fees,

       1000y‚Çê + 997dy‚Çê

The lesson simply postpones the fee adjustment to keep the derivation easier to follow.
