# Step 2 – Understanding the Profit Function

> **Goal of this step**
>
> Before introducing derivatives or optimization, we first need to understand **what exactly we're trying to maximize.**
>
> We already know that we want the **Optimal Amount In**, but what determines whether one trade size is better than another?
>
> The answer is **profit**.
>
> Therefore, before finding the best trade size, we first need a way to describe profit mathematically.

---

# Let's Start With Something Familiar

Suppose we perform an arbitrage.

We start with

```
100 DAI
```

We use that DAI to buy WETH from **AMM A**.

Then we immediately sell that WETH on **AMM B**.

After the entire arbitrage finishes, we receive

```
108 DAI
```

How much profit did we make?

Simple.

```
108 - 100

=

8 DAI
```

No mathematics.

No calculus.

Just basic subtraction.

---

# Let's Give These Numbers Names

Mathematicians dislike writing long sentences repeatedly.

Instead of writing

```
Money I Put Into AMM A
```

they simply give it a variable.

The lesson calls it

```
dyₐ
```

Don't let the notation scare you.

It simply means

> **The amount of token Y that we put into AMM A.**

Nothing more.

Think of it as

```
Trade Size
```

---

Likewise,

instead of writing

```
Money I Got Back
```

they introduce another variable.

```
dy_b
```

This means

> **The amount of token Y we receive after completing the arbitrage.**

Again,

nothing magical.

---

# Our Example Using These Variables

Instead of writing

```
Input

=

100 DAI
```

the lesson writes

```
dyₐ

=

100
```

---

Instead of writing

```
Output

=

108 DAI
```

the lesson writes

```
dy_b

=

108
```

The numbers haven't changed.

Only the names have.

---

# Profit Still Means The Same Thing

Earlier we calculated

```
Profit

=

108

-

100

=

8
```

Now replace the numbers with their variable names.

```
Profit

=

dy_b

-

dyₐ
```

Notice something important.

The mathematics didn't become more complicated.

We simply replaced English names with shorter symbols.

---

# Where Does F(dyₐ) Come From?

Now the lesson suddenly introduces

```
F(dyₐ)
```

This scares many people.

Let's translate it into plain English.

---

Imagine a vending machine.

```
Insert Money

↓

Machine

↓

Receive Snacks
```

Different inputs produce different outputs.

```
₹10

↓

Chips
```

```
₹20

↓

Chocolate
```

The machine's output depends on what we put into it.

Mathematicians call this type of relationship a

> **Function**

because it maps an input to an output.

---

# Arbitrage Behaves Exactly The Same Way

Think of arbitrage as a machine.

```
Trade Size

↓

Arbitrage

↓

Profit
```

Different trade sizes produce different profits.

For example,

```
100 DAI

↓

8 DAI Profit
```

```
500 DAI

↓

32 DAI Profit
```

```
1000 DAI

↓

55 DAI Profit
```

Notice the pattern.

The input changes.

The output changes.

Exactly like a vending machine.

---

# So What Does F Mean?

The letter

```
F
```

is simply the name of our

```
Profit Machine
```

It tells us

> "Given a particular trade size, what profit do we get?"

For example,

```
F(100)

=

8
```

means

> "If I trade 100 DAI, my profit is 8 DAI."

Likewise,

```
F(500)

=

32
```

means

> "If I trade 500 DAI, my profit is 32 DAI."

---

# Why Is It Written As F(dyₐ)?

Remember,

our input variable isn't called

```
100
```

or

```
500
```

The lesson calls it

```
dyₐ
```

Therefore,

instead of writing

```
F(100)
```

the lesson writes

```
F(dyₐ)
```

Read it as

> **"The profit produced when the input amount is `dyₐ`."**

Nothing more.

---

# Understanding The Equation

The lesson writes

```
F(dyₐ)

=

dy_b

-

dyₐ
```

At first glance,

it looks intimidating.

But translate it back into English.

```
Profit

=

Money Received

-

Money Invested
```

That's literally all the equation is saying.

Nothing has changed.

We've simply replaced long descriptions with variable names.

---

# The Big Takeaway

At this point,

we have **not** done any optimization.

We have **not** used calculus.

We have **not** taken derivatives.

We have simply defined three things.

```
dyₐ

=

The amount we put into AMM A.
```

```
dy_b

=

The amount we receive after the arbitrage.
```

```
F(dyₐ)

=

The profit generated by that trade.
```

Everything that follows in the lesson builds on these three simple definitions.

The next question becomes

> **"Which value of `dyₐ` makes `F(dyₐ)` as large as possible?"**

That is where optimization begins.

---

# ✅ Step 2 Complete

At this point we understand:

- Why the lesson introduces `dyₐ`.
- Why it introduces `dy_b`.
- What `F(dyₐ)` represents.
- Why `F(dyₐ) = dy_b - dyₐ` is simply another way of writing:

> **Profit = Money Received − Money Invested.**

We are now ready to understand **why the derivative is introduced** in the next step.
