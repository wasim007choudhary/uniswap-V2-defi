# Step 4 – Where Does `dy_b` Actually Come From?

> **Goal of this step**
>
> Up until now, we've treated `dy_b` as if it were a magic number that simply appears after an arbitrage.
>
> But in reality, `dy_b` is **calculated**.
>
> Before deriving any formulas, we first need to understand **where it comes from** and **what it depends on**.
>
> This step introduces the dependency chain that will be used throughout the rest of the optimization derivation.

---

# Up Until Now...

We've been thinking about arbitrage like this.

```
Trade Size

↓

Arbitrage

↓

Profit
```

Or mathematically,

```
dyₐ

↓

Arbitrage

↓

dy_b
```

It almost looks like

```
dyₐ

↓

dy_b
```

as if the input magically becomes the output.

But that's **not** what actually happens.

---

# Remember How Arbitrage Really Works

Suppose we're arbitraging between two AMMs.

We start with

```
dyₐ
```

which is our input amount.

For example,

```
1000 DAI
```

We **do not** directly receive

```
dy_b
```

Instead,

two completely separate swaps happen.

---

# First Swap

On **AMM A**, we trade

```
DAI

↓

WETH
```

Using the lesson's notation,

```
dyₐ

↓

dxₐ
```

After the first swap,

we no longer own DAI.

We now own

```
dxₐ

(WETH)
```

---

# Second Swap

Now we immediately take that WETH

```
dxₐ
```

and swap it on **AMM B**.

```
WETH

↓

DAI
```

or mathematically,

```
dxₐ

↓

dy_b
```

Now we've returned to DAI,

which completes the arbitrage.

---

# The Complete Flow

Putting both swaps together,

the arbitrage actually looks like this.

```
dyₐ

↓

AMM A

↓

dxₐ

↓

AMM B

↓

dy_b
```

Notice something very important.

There is **no direct path**

from

```
dyₐ

↓

dy_b
```

Instead,

the output from the first swap becomes the input to the second swap.

---

# This Is The Key Observation

What determines

```
dy_b
```

?

Does it depend directly on

```
dyₐ
```

Not exactly.

Instead,

```
dy_b
```

depends on

```
dxₐ
```

But

```
dxₐ
```

itself depends on

```
dyₐ
```

So the real dependency chain is

```
dyₐ

↓

dxₐ

↓

dy_b
```

---

# Think Of It Like Dominoes

Imagine three dominoes.

```
Domino A

↓

Domino B

↓

Domino C
```

Push the first domino.

The second falls.

When the second falls,

the third falls.

The third domino didn't move **because of the first domino directly**.

It moved because

```
First

↓

Second

↓

Third
```

Exactly the same thing happens in arbitrage.

```
dyₐ

↓

dxₐ

↓

dy_b
```

Change the input amount.

↓

The amount of WETH received changes.

↓

The amount of DAI received from the second swap changes.

Everything propagates through the chain.

---

# Another Analogy – Water Tanks

Imagine three water tanks connected by pipes.

```
Tank A

↓

Tank B

↓

Tank C
```

If we pour one extra litre into Tank A,

Tank B changes.

Since Tank B changed,

Tank C also changes.

Tank C never changed **directly** because we poured water into Tank A.

It changed because Tank B changed first.

Again,

our arbitrage behaves exactly the same way.

```
dyₐ

↓

dxₐ

↓

dy_b
```

---

# Why Is This Important?

Earlier we derived

```
F'(dyₐ)

=

dy_b'

-

1
```

Our optimization problem now requires us to compute

```
dy_b'
```

At first glance,

it might seem like we can calculate it directly.

But we can't.

Because

```
dy_b
```

is **not directly determined by**

```
dyₐ
```

Instead,

it is determined by

```
dxₐ
```

which itself depends on

```
dyₐ
```

So we have

```
dyₐ

↓

dxₐ

↓

dy_b
```

---

# This Is Exactly Why The Chain Rule Appears

The Chain Rule is used whenever one quantity depends on another,

which itself depends on another.

That is precisely our situation.

```
dyₐ

↓

dxₐ

↓

dy_b
```

The lesson isn't introducing a brand-new economic concept.

It's simply following the exact path that the tokens take during the arbitrage.

---

# Where Do We Go Next?

Now that we understand the dependency chain,

the next question becomes

> **"How much WETH do we receive from the first swap?"**

To answer that,

we return to something we already know extremely well:

```
x × y = k
```

Using the Constant Product Formula,

we'll derive the expression for

```
dxₐ
```

Once we know

```
dxₐ
```

we can determine

```
dy_b
```

and finally compute

```
dy_b'
```

which is the missing piece needed to calculate the **Optimal Amount In**.

---

# ✅ Step 4 Complete (Conceptual)

At this point we understand:

- Arbitrage consists of **two swaps**, not one.
- We do **not** directly convert `dyₐ` into `dy_b`.
- The output of the first swap (`dxₐ`) becomes the input of the second swap.
- The dependency chain is

```
dyₐ

↓

dxₐ

↓

dy_b
```

- This dependency chain is exactly why the **Chain Rule** will be needed in the mathematical derivation.

The next step begins the actual algebra by deriving **`dxₐ` from the Constant Product Formula (`x × y = k`)**.
