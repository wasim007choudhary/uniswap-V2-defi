# A Real Example

Earlier, we wrote the implementation like this:

```solidity
uint256 x = ...;
uint256 y = ...;
uint256 z = ...;

uint256 amountIn =
    (Math.sqrt(y) - x) / z;
```

The `...` was only used to simplify the explanation.

In a real arbitrage implementation, those values are computed directly from the pool reserves.

For example, suppose we have the following reserves.

```
Pool A

reserveA0 = 1,000 ETH
reserveA1 = 2,000,000 USDC
```

```
Pool B

reserveB0 = 2,050,000 USDC
reserveB1 = 1,000 ETH
```

The arbitrage contract first reads these reserves.

```solidity
(uint112 reserveA0, uint112 reserveA1,) = pairA.getReserves();

(uint112 reserveB0, uint112 reserveB1,) = pairB.getReserves();
```

Then, instead of plugging everything into one gigantic equation, it computes several intermediate values.

```solidity
uint256 feeNumerator = 997;
uint256 feeDenominator = 1000;

uint256 a =
(
    feeNumerator * reserveA1 +
    feeDenominator * reserveB0
) ** 2;

uint256 b =
2 *
feeDenominator *
reserveA0 *
reserveB0 *
(
    feeNumerator * reserveA1 +
    feeDenominator * reserveB0
);

uint256 c =
(feeDenominator * reserveA0 * reserveB0) *
(feeDenominator * reserveA0 * reserveB0)
-
(feeNumerator * feeNumerator) *
reserveA0 *
reserveA1 *
reserveB0 *
reserveB1;
```

After computing these three values, the contract applies the quadratic formula.

```solidity
uint256 discriminant =
    b * b - 4 * a * c;

uint256 amountIn =
(
    Math.sqrt(discriminant)
    -
    b
)
/
(2 * a);
```

This `amountIn` is the mathematically optimal amount to swap.

The bot then executes the arbitrage using that value.

```solidity
swapPoolA(amountIn);

swapPoolB(...);
```

# Why Compute `a`, `b`, and `c`?

Suppose you are solving

```
x² + 5x + 6 = 0
```

Instead of writing

```solidity
(
Math.sqrt(
5 * 5 - 4 * 1 * 6
)
-
5
)
/
(
2 * 1
)
```

you naturally write

```solidity
uint256 a = 1;

uint256 b = 5;

uint256 c = 6;

uint256 answer =
(
Math.sqrt(b * b - 4 * a * c)
-
b
)
/
(
2 * a
);
```

This is exactly the same idea.

The only difference is that, in arbitrage,

- `a`
- `b`
- `c`

are **not fixed numbers**.

They are computed from the liquidity pool reserves.

---

# ⚠️ Very Important

The implementation above is **one valid algebraic form** of the quadratic solution.

Different arbitrage implementations may compute different-looking expressions for `a`, `b`, and `c`.

For example, some implementations use variables such as

```solidity
left
right
k
numerator
denominator
```

instead.

Others simplify or factor the equation before coding it.

Although the code may look different, it is still solving the **same mathematical optimization problem**.

In other words,

> Different code ≠ Different mathematics.

They are simply different ways of implementing the same formula.
