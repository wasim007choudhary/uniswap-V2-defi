# Step F3 – Deriving the Second Fee-Adjusted Swap Equation

> **Goal of this step**
>
> In **Step F2**, we derived the fee-adjusted equation for the first swap:
>
> ```
>             997xₐdyₐ
> dxₐ = ----------------------
>        1000yₐ + 997dyₐ
> ```
>
> Now we'll derive the output of the **second swap**.
>
> Just like before, we'll use the Constant Product Formula, but this time the input token is `dxₐ` and the output token is `dy_b`.
>
> Our goal is to derive
>
> ```
>               997y_bdxₐ
> dy_b = ----------------------------
>         1000x_b + 997dxₐ
> ```

---

# Zoom Into AMM B

Forget AMM A for now.

We're only interested in the second liquidity pool.

Initially,

```
Reserve X = x_b

Reserve Y = y_b
```

The pool still satisfies

```
x_b · y_b = k
```

Exactly the same invariant.

Nothing changes.

---

# What Is Our Input?

From the first swap,

we received

```
dxₐ
```

Those tokens are now sent into AMM B.

However,

just like before,

Uniswap charges a **0.30% fee**.

Therefore,

only

```
997
---- × dxₐ
1000
```

is treated as the **effective input** for pricing.

---

# Reserve Updates

The output token leaves the pool.

Therefore,

```
New Y Reserve

=

y_b - dy_b
```

The input token participates in pricing as

```
             997
x_b + ---------------- × dxₐ
            1000
```

Notice something.

Exactly like Step F2,

the invariant still remains

```
xy = k
```

Only the effective input changes.

---

# Apply The Constant Product Formula

Substitute the new reserves.

```
(
             997
x_b + ---------------- dxₐ
            1000
)

×

(y_b - dy_b)

=

x_by_b
```

This is exactly the same structure as before.

---

# Expand The Left Side

Distribute the multiplication.

```
x_by_b

-

x_bdy_b

          997
+ ---------dxₐy_b

      1000

          997
- ---------dxₐdy_b

      1000

=

x_by_b
```

---

# Cancel Common Terms

Subtract

```
x_by_b
```

from both sides.

Result

```
          997
---------dxₐy_b

      1000

=

x_bdy_b

          997
+

---------dxₐdy_b

      1000
```

Again,

this is exactly the same algebra as the zero-fee derivation.

---

# Factor Out `dy_b`

Both terms on the right contain

```
dy_b
```

Factor it.

```
          997
---------dxₐy_b

      1000

=

dy_b
(
          997
x_b + ---------dxₐ
        1000
)
```

---

# Isolate `dy_b`

Divide both sides.

Result

```
                     997
               y_b --------- dxₐ
                   1000
dy_b = -----------------------------------
                997
         x_b + --------- dxₐ
              1000
```

This is already correct.

We simply clean it up.

---

# Remove The Fraction

Multiply the numerator and denominator by

```
1000
```

The numerator becomes

```
997y_bdxₐ
```

The denominator becomes

```
1000x_b + 997dxₐ
```

Final result

```
              997y_bdxₐ
dy_b = ----------------------------
        1000x_b + 997dxₐ
```

This is the fee-adjusted version of the second swap equation.

---

# Compare With The Zero-Fee Formula

Without fees,

```
            y_bdxₐ
dy_b = ----------------
         x_b + dxₐ
```

With Uniswap V2's fee,

```
              997y_bdxₐ
dy_b = ----------------------------
        1000x_b + 997dxₐ
```

The structure is identical.

Only the effective input changes.

---

# Putting Both Swaps Together

We now have the fee-adjusted equations for **both** swaps.

### First Swap

```
             997xₐdyₐ
dxₐ = ----------------------
        1000yₐ + 997dyₐ
```

### Second Swap

```
              997y_bdxₐ
dy_b = ----------------------------
        1000x_b + 997dxₐ
```

Notice something.

We're back to exactly the same dependency chain we had before.

```
dyₐ

↓

dxₐ

↓

dy_b
```

The only difference is that each swap now includes the fee-adjusted terms.

---

# Looking Ahead

The next step is **identical** to what we did in the zero-fee derivation.

We'll substitute the first equation into the second,

eliminating the intermediate variable

```
dxₐ
```

After substitution,

the entire arbitrage will once again depend on only one variable:

```
dyₐ
```

From there,

we'll simplify,

differentiate,

and eventually derive the production-ready Optimal Amount In formula.
