# Step F4 – Eliminating the Intermediate Variable (`dxₐ`)

> **Goal of this step**
>
> We now have the fee-adjusted equations for both swaps.
>
> **First swap**
>
> ```
>             997xₐdyₐ
> dxₐ = ----------------------
>        1000yₐ + 997dyₐ
> ```
>
> **Second swap**
>
> ```
>               997y_bdxₐ
> dy_b = ----------------------------
>         1000x_b + 997dxₐ
> ```
>
> The problem is that `dy_b` still depends on the intermediate variable `dxₐ`.
>
> Our objective is to eliminate `dxₐ` so the entire arbitrage depends only on the initial input `dyₐ`.

---

# The Dependency Chain

Currently,

```
dyₐ

↓

dxₐ

↓

dy_b
```

For optimization,

we eventually need

```
dyₐ

↓

dy_b
```

That means `dxₐ` must disappear.

---

# The Tool We'll Use

Nothing new is introduced here.

We'll use the same technique as the zero-fee derivation:

> **Substitution**

Since we already know

```
             997xₐdyₐ
dxₐ = ----------------------
        1000yₐ + 997dyₐ
```

we simply replace every occurrence of `dxₐ` inside the second equation.

---

# Begin The Substitution

Start with

```
               997y_bdxₐ
dy_b = ----------------------------
         1000x_b + 997dxₐ
```

Replace every `dxₐ`.

The numerator becomes

```
                    997xₐdyₐ
               ----------------------
                  1000yₐ + 997dyₐ
997y_b × --------------------------------
```

The denominator becomes

```
                          997xₐdyₐ
                     ----------------------
                        1000yₐ + 997dyₐ
1000x_b + 997 × --------------------------------
```

Putting everything together,

```
                        997xₐdyₐ
                   ----------------------
                     1000yₐ + 997dyₐ
dy_b = 997y_b ------------------------------------------
                             997xₐdyₐ
                        ----------------------
                          1000yₐ + 997dyₐ
         1000x_b + 997 ----------------------
```

At first glance,

this looks much uglier than the zero-fee version.

Don't worry.

It's still the exact same idea.

The extra complexity comes entirely from the fee factors.

---

# What Makes This Expression Ugly?

Notice the denominator.

It contains

```
1000x_b

+

a fraction
```

Just like before,

we cannot add these directly.

They need a common denominator.

This is identical to what we did during the zero-fee derivation.

---

# Give Both Terms A Common Denominator

Rewrite

```
1000x_b
```

as

```
1000x_b(1000yₐ + 997dyₐ)
-----------------------------------
      1000yₐ + 997dyₐ
```

Now both denominator terms share the same denominator.

That allows us to combine them into one fraction.

---

# Combine The Denominator

After combining,

the denominator becomes

```
1000x_b(1000yₐ + 997dyₐ)

+

997²xₐdyₐ
-----------------------------------
      1000yₐ + 997dyₐ
```

Notice something new.

The term

```
997²
```

appears naturally.

We didn't invent it.

It appeared because

```
997

×

997

=

997²
```

This is exactly why production arbitrage formulas often contain `997²`.

---

# Divide By A Fraction

Our equation now has the form

```
A
---
B/C
```

Just like before,

dividing by a fraction means multiplying by its reciprocal.

So we multiply by

```
          1000yₐ + 997dyₐ
--------------------------------------------------
1000x_b(1000yₐ + 997dyₐ) + 997²xₐdyₐ
```

---

# A Convenient Cancellation

The factor

```
1000yₐ + 997dyₐ
```

appears in both places.

It cancels completely.

After cancellation,

we obtain a much cleaner expression.

```
                  997²xₐy_bdyₐ
dy_b = ----------------------------------------------------
        1000x_b(1000yₐ + 997dyₐ)
        +
        997²xₐdyₐ
```

We've successfully removed the intermediate variable.

---

# Expand The Denominator

Expand

```
1000x_b(1000yₐ + 997dyₐ)
```

Result

```
1000²x_byₐ

+

1000·997x_bdyₐ
```

Substitute this back.

```
                  997²xₐy_bdyₐ
dy_b = --------------------------------------------------------
        1000²x_byₐ

        +

1000·997x_bdyₐ

        +

997²xₐdyₐ
```

Finally,

factor out the common

```
dyₐ
```

from the last two terms.

```
                         997²xₐy_bdyₐ
dy_b = ---------------------------------------------------------
        1000²x_byₐ

        +

(1000·997x_b + 997²xₐ)dyₐ
```

This is the fee-adjusted version of the simplified arbitrage equation.

---

# Compare With The Zero-Fee Result

Without fees,

we obtained

```
                xₐy_bdyₐ
dy_b = -----------------------------------------
        x_byₐ + (xₐ + x_b)dyₐ
```

With Uniswap V2 fees,

we now have

```
                         997²xₐy_bdyₐ
dy_b = ---------------------------------------------------------
        1000²x_byₐ

        +

(1000·997x_b + 997²xₐ)dyₐ
```

The overall structure is identical.

The only difference is that the fee constants now appear throughout the equation.

---

# Why Is This Important?

This is the first equation that describes the **entire arbitrage** while accounting for trading fees.

Notice that

```
dxₐ
```

has disappeared completely.

Everything now depends on only one variable:

```
dyₐ
```

Exactly what we need for optimization.

---

# Looking Ahead

From here onward,

the derivation follows the exact same roadmap as before.

```
Simplified Arbitrage Function

↓

Differentiate

↓

Set

dy_b' = 1

↓

Solve

↓

Optimal Amount In
```

The calculus doesn't change.

Only the constants become larger because the fee factors propagate through the algebra.

---

# ✅ Step F4 Complete

At this point we understand:

- Why `dxₐ` must be eliminated.
- How substitution is applied to the fee-adjusted swap equations.
- Why a common denominator is required.
- Why the factor `997²` naturally appears.
- How the intermediate variable disappears completely.
- How the arbitrage simplifies to

```
                         997²xₐy_bdyₐ
dy_b = ---------------------------------------------------------
        1000²x_byₐ

        +

(1000·997x_b + 997²xₐ)dyₐ
```

This is the fee-adjusted arbitrage function that will be differentiated in the next step to derive the production-ready **Optimal Amount In** formula.
