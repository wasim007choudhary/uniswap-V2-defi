# Step 3 – Why Do We Suddenly Take a Derivative?

> **Goal of this step**
>
> Up until now, we've only defined our profit function.
>
> ```
> F(dyₐ) = dy_b - dyₐ
> ```
>
> Now our objective changes.
>
> We no longer want to simply calculate profit.
>
> We want to **find the trade size that produces the maximum profit.**
>
> To do that, calculus introduces one very powerful tool:
>
> **The Derivative.**

# But How Do We Know We've Reached ★?

Imagine you're climbing a mountain.

```
          ★
         /\
        /  \
       /    \
______/      \______
```

Now imagine you're blindfolded.

You cannot see the peak.

How do you know whether you should keep walking?

You ask one very simple question.

> **"If I take one tiny step forward, do I go up or down?"**

That's it.

---

## Case 1 — You're Still Climbing

```
      /
     /
____/
```

Take one tiny step.

You go higher.

That means

> **Keep walking.**

You haven't reached the top yet.

---

## Case 2 — You've Reached The Peak

```
        ★
       /\
```

Take one tiny step.

Nothing gets higher anymore.

You're at the highest point.

---

## Case 3 — You've Passed The Peak

```
        ★
         \
          \
```

Take one tiny step.

You go lower.

Oops.

You've already passed the top.

---

# Notice What We're Measuring

We never asked

> "How tall is the mountain?"

Instead we asked

> **"If I move a tiny bit, does my height increase or decrease?"**

That is an extremely important distinction.

We're measuring

```
Change
```

not

```
Height
```

Whenever we increase the trade by a tiny amount,

only three things can happen.

## Possibility 1

# This Is Exactly What A Derivative Measures

A derivative answers one simple question.

> **"If I increase my input by a tiny amount, how does my output change?"**

Notice carefully.

It does **not** ask

> "What is my profit?"

Instead it asks

> **"How is my profit changing?"**

Those are completely different questions.

---

# Think About Driving

Suppose someone asks

> "How far have you traveled?"

That's one question.

Now suppose they ask

> "How fast are you moving right now?"

That's a completely different question.

```
Distance

≠

Speed
```

Speed measures

> **How quickly your distance changes.**

Likewise,

```
Profit

≠

Derivative of Profit
```

The derivative measures

> **How quickly profit changes as we change the trade size.**

---

# Why Is This Useful?

Because the highest point always has a special property.

Before the peak,

profit increases.

```
Derivative > 0
```

---

Exactly at the peak,

profit stops increasing.

```
Derivative = 0
```

---

After the peak,

profit decreases.

```
Derivative < 0
```

This is true not only in arbitrage,

but in almost every smooth optimization problem.

---

# So Why Does Every Optimization Problem Set The Derivative To Zero?

Not because mathematicians enjoy derivatives.

Not because DeFi requires calculus.

Simply because

> **The highest point of a smooth curve occurs where the slope becomes zero.**

That is exactly what we're searching for.

---

# Now Let's Differentiate Our Profit Function

We already know

```
F(dyₐ)

=

dy_b

-

dyₐ
```

Remember the English translation.

```
Profit

=

Money Received

-

Money Invested
```

Nothing scary.

---

# We Now Ask A Different Question

Previously we asked

> **"How much profit do I make?"**

Now we ask

> **"How does my profit change if I increase my trade by a tiny amount?"**

Notice,

we changed the **question**, not the formula.

---

# What Does The Little ' Mean?

The little apostrophe

```
'
```

(pronounced **prime**)

means

> **Rate of Change**

So

```
F'
```

does **not** mean

```
A different function.
```

It means

```
The rate at which profit changes.
```

Likewise,

```
dy_b'
```

means

```
How fast the money received changes.
```

---

# Differentiate Both Sides

Start with

```
F(dyₐ)

=

dy_b

-

dyₐ
```

Take the derivative of both sides.

The left side becomes

```
F'(dyₐ)
```

Easy.

---

The right side is

```
dy_b

-

dyₐ
```

The derivative of a subtraction is simply

```
Derivative of first

-

Derivative of second
```

So we get

```
F'(dyₐ)

=

dy_b'

-

(dyₐ)'
```

---

# Why Does (dyₐ)' Become 1?

Remember,

our input variable is

```
dyₐ
```

Everything is being measured **with respect to** `dyₐ`.

In plain English,

we're asking

> **"If dyₐ changes by one tiny unit..."**

then

> **"How much do the other quantities change?"**

Now think about the input itself.

If the trade increases

```
100 DAI

↓

101 DAI
```

How much did the input increase?

```
1 DAI
```

If it goes

```
500

↓

501
```

Again,

it increased by

```
1
```

Always.

The input changes one-for-one with itself.

Therefore,

```
(dyₐ)' = 1
```

This is the same rule as

```
d(x)
/---
dx

=

1
```

because

> **A variable changes one-for-one with itself.**

---

# Substitute It Back

We had

```
F'(dyₐ)

=

dy_b'

-

(dyₐ)'
```

Replace

```
(dyₐ)'
```

with

```
1
```

Result

```
F'(dyₐ)

=

dy_b'

-

1
```

That's it.

No hidden tricks.

No skipped steps.

---

# What Does This Mean Intuitively?

Remember,

profit is

```
Money Received

-

Money Invested
```

When we slightly increase the trade,

two things happen.

---

## Money Invested

If we spend one extra DAI,

our investment increases by exactly one DAI.

That's why its derivative is

```
1
```

---

## Money Received

This is more interesting.

Does one extra DAI invested always produce one extra DAI received?

No.

Because of

- Slippage
- AMM pricing
- Constant Product Formula

Sometimes

```
Spend 1 extra DAI

↓

Receive 1.05 extra DAI
```

Sometimes

```
Spend 1 extra DAI

↓

Receive 0.98 extra DAI
```

Sometimes

```
Spend 1 extra DAI

↓

Receive 0.80 extra DAI
```

The rate keeps changing.

That's exactly why

```
dy_b'
```

is **not** a constant.

It depends on the current trade size.

---

# Why Do We Set The Derivative Equal To Zero?

Remember our mountain.

At the highest point,

the slope is zero.

So we write

```
F'(dyₐ)

=

0
```

Substitute the equation we just derived.

```
dy_b'

-

1

=

0
```

Move the `1` to the other side.

```
dy_b'

=

1
```

---

# What Does This Result Actually Mean?

This is much more than just algebra.

It tells us

> **The optimal trade occurs when one additional unit invested returns exactly one additional unit.**

Think about the three possibilities.

---

### Before The Optimum

```
Spend 1 extra DAI

↓

Receive 1.20 extra DAI
```

Worth increasing the trade.

---

### At The Optimum

```
Spend 1 extra DAI

↓

Receive exactly 1 extra DAI
```

You've reached the maximum profit.

---

### After The Optimum

```
Spend 1 extra DAI

↓

Receive only 0.90 extra DAI
```

Now every additional DAI reduces your overall profit.

You've gone too far.
