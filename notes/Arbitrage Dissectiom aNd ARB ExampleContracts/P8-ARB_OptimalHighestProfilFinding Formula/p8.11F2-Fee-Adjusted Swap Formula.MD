# Step F2 – Deriving the Fee-Adjusted Swap Formula From the Constant Product Formula

> **Goal of this step**
>
> In **Step F1**, we learned that Uniswap V2 charges a **0.3% trading fee**, meaning only **99.7% of the input** participates in price calculation.
>
> In this step, we'll derive the actual Uniswap V2 swap equation from the **Constant Product Formula**, just as we did in the zero-fee derivation.
>
> The goal is to prove
>
> ```
>            997xdy
> dx = ----------------------
>       1000y + 997dy
> ```
>
> directly from the AMM invariant.

---

# Does The Constant Product Formula Change?

The first question many people ask is

> **"Since Uniswap charges a fee, does the Constant Product Formula change?"**

The answer is

> **No.**

The invariant is still

```
xy = k
```

The AMM is still a Constant Product Market Maker.

Nothing changes here.

---

# Then What Changes?

The pool still satisfies

```
xy = k
```

The reserve updates are still

```
New X Reserve

=

x - dx
```

```
New Y Reserve

=

y + ?
```

Notice the question mark.

This is where the fee changes everything.

---

# The Zero-Fee Derivation

Previously,

if the user supplied

```
dy
```

the pool behaved as though it received

```
dy
```

Therefore,

the new reserve became

```
y + dy
```

leading to

```
(x - dx)(y + dy) = xy
```

---

# The Fee-Adjusted World

Now suppose the user sends

```
dy
```

to the pool.

The pool receives all of it.

However,

only

```
997
---- × dy
1000
```

is treated as the **effective input** for pricing.

Therefore,

instead of writing

```
y + dy
```

inside the pricing equation,

we write

```
             997
y + ---------------- × dy
            1000
```

Notice something very important.

The **pricing equation** uses the effective input.

The **pool balance** still increases by the full amount.

This distinction is what creates the LP fee.

---

# Apply The Constant Product Formula

Exactly like before,

the invariant is

```

×

(New Y)

Old X × Old Y
```

Substitute the reserves.

```

                1000

Nothing else has changed.

# Isolate `dx`

Divide both sides by

                    997

This is already the correct equation.

The only thing left is cleaning it up.

---

# Remove The Fraction

Multiply the numerator and denominator by

```
1000
```

The numerator becomes

```
997xdy
```

The denominator becomes

```
1000y + 997dy
```

The final result is

```
            997xdy
dx = ----------------------
       1000y + 997dy
```

Exactly the formula implemented by Uniswap V2.

---

# Compare With The Zero-Fee Formula

Without fees,

```
          xdy
dx = -------------
       y + dy
```

With fees,

```
            997xdy
dx = ----------------------
       1000y + 997dy
```

The structure hasn't changed.

Only the input amount has been adjusted to account for the trading fee.

Everything we've done so far in the arbitrage derivation still applies.

From this point onward,
