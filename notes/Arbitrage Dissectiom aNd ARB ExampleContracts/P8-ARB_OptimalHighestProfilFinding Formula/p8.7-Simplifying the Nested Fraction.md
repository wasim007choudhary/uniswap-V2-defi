# Step 7 ‚Äì Simplifying the Nested Fraction

> **Goal of this step**
>
> At the end of **Step 6**, we successfully eliminated the intermediate variable `dx‚Çê` by substitution.
>
> However, the resulting equation contained a **fraction inside another fraction**, making it difficult to analyze or differentiate.
>
> The purpose of this step is **not to introduce new concepts**, but simply to use algebra to transform that messy equation into one clean function of `dy‚Çê`.
>
> Once that's done, the arbitrage output (`dy_b`) will depend on only a single variable, allowing us to apply calculus in the next step.

# Where We Left Off

                      x‚Çêdy‚Çê

At first glance,

Fortunately,

Everything that follows is simply algebra.

# Notice The Problem

The equation contains

> **a fraction inside another fraction.**

This type of expression is called a **complex fraction** or **nested fraction**.

Our goal is simply to remove it.

---

# Let's Give The Inner Fraction A Name

To make the equation easier to read,

pretend

```
          x‚Çêdy‚Çê
A = ----------------
      y‚Çê + dy‚Çê
```

Then the equation becomes

```
          y_bA
dy_b = -----------
        x_b + A
```

This doesn't change the mathematics.

It simply reduces visual clutter.

---

# The Real Problem Is The Denominator

The numerator is already simple.

The denominator is

```
x_b + A
```

Substituting `A` back,

we obtain

```
             x‚Çêdy‚Çê
x_b + ----------------
         y‚Çê + dy‚Çê
```

To add these,

they must have a common denominator.

---

# Give `x_b` The Same Denominator

Remember,

any number can be written as a fraction.

For example,

```
      5 √ó 10
5 = ----------
        10
```

The value doesn't change because we've multiplied the numerator and denominator by the same quantity.

Using the same idea,

```
      x_b(y‚Çê + dy‚Çê)
x_b = -------------------
         y‚Çê + dy‚Çê
```

Now both fractions have the same denominator.

---

# Combine The Fractions

Since both fractions now have

```
y‚Çê + dy‚Çê
```

as the denominator,

we simply add the numerators.

Result

```
x_b(y‚Çê + dy‚Çê) + x‚Çêdy‚Çê
--------------------------------
         y‚Çê + dy‚Çê
```

The denominator has now become a **single fraction**.

---

# Substitute It Back

Replacing the denominator,

```
                         x‚Çêdy‚Çê
                    ----------------
                      y‚Çê + dy‚Çê
dy_b = y_b --------------------------------------
               x_b(y‚Çê + dy‚Çê) + x‚Çêdy‚Çê
               ----------------------
                   y‚Çê + dy‚Çê
```

Although it still looks complicated,

the next step makes everything much cleaner.

---

# Divide By A Fraction

Recall one of the most important algebra rules.

```
      a
------------
      b
      -
      c

      c
= a √ó -
      b
```

In words,

> **Dividing by a fraction is the same as multiplying by its reciprocal.**

Applying that rule,

```
             x‚Çêdy‚Çê
        ----------------
          y‚Çê + dy‚Çê

            y‚Çê + dy‚Çê
√ó -------------------------------
   x_b(y‚Çê + dy‚Çê) + x‚Çêdy‚Çê
```

---

# A Convenient Cancellation

Notice that

```
y‚Çê + dy‚Çê
```

appears

- once in the numerator,
- once in the denominator.

Since they're identical,

they cancel.

The equation immediately becomes

```
                x‚Çêy_bdy‚Çê
dy_b = --------------------------------
        x_b(y‚Çê + dy‚Çê) + x‚Çêdy‚Çê
```

The complex fraction is gone.

---

# Expand The Denominator

Expand

```

```

```

```

Substitute this back.

```
                x‚Çêy_bdy‚Çê
dy_b = ------------------------------------
        x_by‚Çê + x_bdy‚Çê + x‚Çêdy‚Çê
```

---

# Factor The Common Term

Notice that

```

x‚Çêdy‚Çê
```

```
dy‚Çê
```

Factor it out.

```
                x‚Çêy_bdy‚Çê
dy_b = -----------------------------------------
        x_by‚Çê + (x‚Çê + x_b)dy‚Çê
```

This is the clean expression we wanted.

---

# Why Is This Form Better?

Previously,

our equation contained

- an intermediate variable (`dx‚Çê`)
- nested fractions
- unnecessary complexity.

Now,

everything has been reduced to a single function.

```
                x‚Çêy_bdy‚Çê
dy_b = -----------------------------------------
        x_by‚Çê + (x‚Çê + x_b)dy‚Çê
```

Most importantly,

```

```

The reserve values

- `x‚Çê`
- `y‚Çê`
- `x_b`
- `y_b`

The only variable remaining is

```

```

Exactly what we need for optimization.
