# Step F6 – Solving for the Optimal Arbitrage Input

> **Goal of this step**
>
> In **Step F5**, we derived the condition for maximum profit:
>
> \[
> \boxed{
> \frac{
> f^2x_ax_by_ay_b
> }
> {
> \left(
> x_by_a+
> (fx_b+f^2x_a)r
> \right)^2
> }
> =
> 1
> }
> \]
>
> The only unknown is
>
> \[
> r,
> \]
>
> which represents the trader's initial input.
>
> Our objective is to solve this equation algebraically.

---

# Step 1 — Remove The Denominator

Multiply both sides by

\[
\left(
x_by_a+
(fx_b+f^2x_a)r
\right)^2.
\]

This gives

\[
\boxed{
f^2x_ax_by_ay_b
=
\left(
x_by_a+
(fx_b+f^2x_a)r
\right)^2
}
\]

Now the denominator has disappeared.

---

# Step 2 — Remove The Square

Since every reserve is positive,

and the optimal input must also be positive,

we take the **positive square root** of both sides.

The left-hand side becomes

\[
\sqrt{
f^2x_ax_by_ay_b
}.
\]

Because

\[
f>0,
\]

we can simplify

\[
\sqrt{f^2}=f.
\]

Therefore,

\[
\boxed{
f\sqrt{x_ax_by_ay_b}
=
x_by_a+
(fx_b+f^2x_a)r
}
\]

Notice how the square disappears.

---

# Why Do We Choose The Positive Root?

Taking the square root technically gives

\[
\pm
f\sqrt{x_ax_by_ay_b}.
\]

However,

the negative solution would produce

a negative trade size,

which has no physical meaning.

Therefore,

we keep only the positive root.

---

# Step 3 — Move The Constant Term

Subtract

\[
x_by_a
\]

from both sides.

\[
\boxed{
f\sqrt{x_ax_by_ay_b}
-
x_by_a
=
(fx_b+f^2x_a)r
}
\]

Now every term containing

\[
r
\]

is isolated on one side.

---

# Step 4 — Divide By The Coefficient

Divide both sides by

\[
fx_b+f^2x_a.
\]

This isolates

\[
r.
\]

\[
\boxed{
r
=
\frac{
f\sqrt{x_ax_by_ay_b}
-
x_by_a
}
{
fx_b+f^2x_a
}
}
\]

This is the optimal arbitrage input expressed in terms of the generic fee multiplier

\[
f.
\]

---

# Factor The Denominator

Notice that

both denominator terms contain

\[
f.
\]

Factor it out.

\[
fx_b+f^2x_a
=
f(x_b+fx_a).
\]

Substitute back.

\[
\boxed{
r
=
\frac{
f\sqrt{x_ax_by_ay_b}
-
x_by_a
}
{
f(x_b+fx_a)
}
}
\]

This form is often preferred because it makes the dependence on the fee multiplier easier to see.

---

# Compare With The Zero-Fee Formula

Without fees,

\[
\boxed{
r
=
\frac{
\sqrt{x_ax_by_ay_b}
-
x_by_a
}
{
x_a+x_b
}
}
\]

With fees,

\[
\boxed{
r
=
\frac{
f\sqrt{x_ax_by_ay_b}
-
x_by_a
}
{
f(x_b+fx_a)
}
}
\]

Notice the structure.

The derivation is almost identical.

The only difference is that the fee multiplier appears naturally in the final coefficients.

---

# What Does This Formula Mean?

Every quantity except

\[
r
\]

is already known.

The arbitrage bot reads

- \(x_a\)
- \(y_a\)
- \(x_b\)
- \(y_b\)

from the two liquidity pools.

The fee multiplier

\[
f
=
\frac{997}{1000}
\]

is fixed by the protocol.

Therefore,

the optimal input can be computed immediately,

without trial and error.
