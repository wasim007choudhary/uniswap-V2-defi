# Why Doesn't Alice Become `msg.sender`?

Because every contract call creates a new execution context.

When Alice calls:

```text id="98yj0d"
Router.swap(...)
```

Inside the Router:

```text id="hkptoz"
msg.sender

=

Alice
```

But then the Router calls:

```text id="2sxh1j"
ERC20.transferFrom(...)
```

Inside the ERC20 contract:

```text id="dfqqxf"
msg.sender

=

Router
```

The caller changed.

That was another major realization.

# Then Another Question

I asked:

Initially I thought:

> "Doesn't the Router eventually run out of ETH?"

Fortunately,

---

# Who Pays?

Only one account pays gas.

The account that started the transaction.

In our example:

```text id="9gxyax"
Alice
```

Alice signs the transaction.

Alice pays the gas.

---

# Does The Router Ever Pay?

No.

The Router never owns ETH for paying transaction fees.

It simply executes code inside Alice's transaction.

Think of the Router like a machine.

Machines don't pay electricity bills.

The person using the machine does.

# Then I Asked Something Else

I wondered:

> **"Can the Router go rogue and steal everyone's tokens?"**

This was probably my biggest security concern.

# The Answer

For the **official Uniswap V2 Router**:

**No.**

The Router is a deployed smart contract.

Once deployed,

its code cannot be modified.

---

No.

Because the deployed Router cannot change.

Its bytecode is immutable.

---

# Final Summary

`transferFrom()` is the foundation of delegated token transfers in ERC20.

Inside Uniswap V2, it enables the Router to move a user's tokens **only because the user explicitly granted permission beforehand**.

The function enforces two independent security checks:

1. **Allowance Check**

   * Has the owner granted sufficient permission to the spender?

2. **Balance Check**

   * Does the owner actually possess the requested amount?

Even an unlimited allowance (`type(uint256).max`) cannot bypass the owner's token balance.

The Router acts only as an authorized spender, while the owner's balance remains the ultimate limit on how many tokens can be transferred.
