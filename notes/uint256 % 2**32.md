# `2**32` & Modulo — Complete Discussion

>You will see the line in UniswapV2Pair _update() you will encounter this line and if you ever wonder why that is done, see here why 2**32 is done and why?

---

## What is `%`?

`%` is the modulo operator. It returns the remainder after division.

A % B = remainder when A is divided by B

10 % 3 = 1    (10 ÷ 3 = 3, remainder 1)
100 % 32 = 4  (100 ÷ 32 = 3, remainder 4)

---

## Uniswap's Line

uint32 blockTimestamp = uint32(block.timestamp % 2**32);

---

## Why `2**32`?

uint32 holds values from 0 to 2^32 - 1 (0 to 4,294,967,295).

block.timestamp is uint256. It grows forever. Currently ~1.7 billion. Eventually it will exceed 2^32 (year 2106).

Modulo 2^32 wraps the value so it always fits in uint32.

---

## Why 2**32 and NOT 2**32 - 1?

% N gives results 0 to N-1.

% 2^32        → results 0 to 2^32 - 1  ✅ Full uint32 range
% (2^32 - 1)  → results 0 to 2^32 - 2  ❌ Loses max value

% (2^32 - 1) would NEVER return 2^32 - 1. The max uint32 value would be impossible.

---

## Example: 5000 % 2**32

5000 % 4,294,967,296 = 5000

When the left number is smaller than the modulus, the result is just the left number. No wrapping happens.

---

## Example: 2^32 + 1

4,294,967,297 % 2^32
= (2^32 + 1) % 2^32
= 1

One full cycle plus 1 extra → wraps to 1.

---

## The Key Property

No matter how HUGE the left number is:

X % 2^32  is ALWAYS between 0 and 2^32 - 1

Even (2^256 - 1) % 2^32 fits in uint32. The remainder is always less than 2^32. Always.

---

## Is block.timestamp uint256?

YES. block.timestamp is uint256.

---

## Is 2**32 uint256?

No. 2**32 alone is a small integer literal. Solidity treats integer literals as their own type that can fit into any uint. When used with block.timestamp (a uint256), the compiler automatically upscales 2**32 to uint256 for the modulo operation.

The result is typed uint256, but the VALUE is always < 2^32, so casting to uint32 is safe.

---

## Visual: The Wrap

block.timestamp values:

0           → % 2^32 = 0
2^32 - 1    → % 2^32 = 2^32 - 1  (max)
2^32        → % 2^32 = 0  (wraps!)
2^32 + 1    → % 2^32 = 1  (wraps!)
2^32 + 100  → % 2^32 = 100 (wraps!)

---

## Do You Need % 2**32 in Solidity ^0.8.0?

No. uint32(block.timestamp) alone works. Solidity ^0.8.0 safely truncates. The modulo was for older Solidity versions.

---

## uint32(...) vs % 2**32

| Part | What It Does |
|------|--------------|
| uint32(...) | Casts to uint32 type |
| % 2**32 | Wraps value to 0 to 2^32-1 range FIRST |

Both together ensure the value always fits. In ^0.8.0, just the cast is enough.

---

## Summary

- X % 2^32 is always between 0 and 2^32 - 1
- Fits in uint32 perfectly — always
- % (2^32 - 1) would lose the max value — wrong
- Small numbers like 5000 don't wrap
- Large numbers like 2^32 + 1 wrap to 1
- On ^0.8.0, uint32(block.timestamp) alone is fine
